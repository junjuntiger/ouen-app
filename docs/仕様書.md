## 3. 各画面の仕様

### SCR-01 ログイン画面

**目的：** SMS認証でユーザーを識別する

**表示要素：**
- アプリロゴ・タイトル「おーえん」
- 電話番号入力フィールド（+81 プレフィックス表示）
- SMSを送るボタン（reCAPTCHA invisible）
- 認証コード入力フィールド（SMS送信後に表示）
- ログインボタン

**バリデーション：**
- 電話番号：11桁（0から始まる・ハイフン除去して検証）
- 認証コード：6桁数字

**エラー表示：**
- 無効な電話番号：「正しい電話番号を入力してください」
- コード不一致：「認証コードが正しくありません」
- 送信失敗：「SMSの送信に失敗しました。時間をおいて再試行してください」

---

### SCR-02 プロフィール登録画面

**目的：** 初回ログイン時にプロフィールを作成する

**表示要素：**
- 名前（必須）
- 職業・活動内容（必須）
- 地域（必須）
- ひとこと（任意・100文字以内）
- 定価メニュー（1件以上必須）
  - メニュー名（例：抹茶体験）
  - 金額（円）
  - 追加ボタン / 削除ボタン
- 登録ボタン

**完了後の遷移：** SCR-03 ホーム

---

### SCR-03 ホーム画面

**目的：** OP残高の確認と最新おーえん情報の閲覧

**表示要素：**
- OP残高カード（大きく中央表示）
  - 現在のOP数値（アニメーション付き）
  - 「おーえんする」ボタン（大）
- 承認待ちバナー（pending取引がある場合のみ）
  - 件数表示・タップで詳細
- おーえんタイムライン
  - 直近20件の取引（リアルタイム更新）
  - 送受信者名・金額・OP・日時

**データ取得：** Firestore onSnapshot（リアルタイム）

---

### SCR-04 おーえんする画面

**目的：** 相手を選んで取引を行う

**フロー（ステップ形式）：**

```
Step 0: 相手を探す
  ├─ タブ①：QRスキャン（カメラ起動 → userId取得）
  └─ タブ②：メンバー検索（名前・地域で絞り込み）
        │
        ▼
Step 1: メニュー選択
  - 相手のプロフィールカード表示
  - 定価メニュー一覧からタップで選択
        │
        ▼
Step 2: 支払い金額入力
  - 選択メニューと定価を表示
  - 支払い金額入力（定価以上のみ）
  - OP自動計算表示（リアルタイム）
        │
        ▼
Step 3: 確認
  - 取引内容サマリー表示
  - 送信ボタン → Firestoreにpending作成
        │
        ▼
Step 4: 完了
  - 「おーえんを送りました！」表示
  - 相手の承認待ち状態を表示
```

**バリデーション：**
- 支払い金額 ≥ 定価（満たさない場合エラー表示）
- 自分自身への送信不可

---

### SCR-05 メンバー一覧画面

**目的：** 登録メンバーを一覧で確認する

**表示要素：**
- 検索バー（名前・地域で絞り込み）
- メンバーカード一覧
  - プロフィール写真（未設定時はイニシャル）
  - 名前・職業・地域
  - OP残高
  - タップでプロフィール詳細 → おーえんする画面へ

**ソート：** OP残高の降順（デフォルト）

---

### SCR-06 マイページ画面

**目的：** 自分の情報確認と設定

**表示要素：**
- プロフィールカード（名前・職業・地域・ひとこと）
- QRコード（自分のuserId エンコード）
- OP残高
- 取引履歴タブ
  - 送った / 受け取ったで切り替え
  - 日付・相手名・金額・OP
- プロフィール編集ボタン
- ログアウトボタン

## 4. 共通コンポーネント

| コンポーネント | 役割 |
|---|---|
| BottomNav | 画面下部のタブナビゲーション（ホーム / おーえん / メンバー / マイページ） |
| Layout | ページ共通レイアウト（ヘッダー + コンテンツ + BottomNav） |
| OPBadge | OP数値を強調表示するバッジ |
| TransactionCard | タイムライン・履歴の1件表示カード |
| LoadingSpinner | ローディング中の表示 |
