# セキュリティ設計書

**プロジェクト名：** おーえん（OUEN-APP）  
**作成日：** 2026-06-25

---

## 1. 対策済み項目

### 1-1. 認証・認可

| 対策 | 内容 | 実装箇所 |
|---|---|---|
| SMS認証 | 電話番号 + ワンタイムコードで本人確認 | Firebase Phone Auth |
| reCAPTCHA | ボット・自動化ツールによる不正ログインを防止 | Login画面（invisible） |
| IDトークン検証 | APIリクエスト時にFirebase IDトークンをサーバー側で検証 | Vercel Functions |
| セッション管理 | Firebase SDKによる自動トークンリフレッシュ | SDK内部処理 |

### 1-2. データアクセス制御

| 対策 | 内容 |
|---|---|
| Firestoreセキュリティルール | 未認証アクセスを全拒否 |
| ユーザーデータ保護 | 自分のドキュメントのみ書き込み可 |
| 取引データ保護 | fromUser / toUser のみ読み取り可 |
| OP残高の改ざん防止 | クライアントからの直接op更新を禁止（API経由のみ） |
| 自己取引防止 | fromUserId == toUserId の取引作成をルールで拒否 |
| 金額バリデーション | paid >= price をFirestoreルールで強制 |

### 1-3. 入力バリデーション

| 対策 | 内容 | 実施場所 |
|---|---|---|
| 電話番号形式チェック | 正規表現で日本の番号形式を確認 | フロントエンド |
| 金額の正数チェック | 0以下・小数を拒否 | フロントエンド + Firestoreルール |
| 文字数制限 | ひとこと100文字・メニュー名50文字 | フロントエンド |
| 画像サイズ制限 | 5MB以上はアップロード拒否（今後追加） | フロントエンド + API |

### 1-4. 通信セキュリティ

| 対策 | 内容 |
|---|---|
| HTTPS強制 | Vercelはデフォルトですべての通信をHTTPS化 |
| Firebase SSL | FirestoreやAuth通信はFirebase SDKがSSL処理 |
| CORS設定 | Vercel FunctionsのCORSを本番ドメインのみ許可 |

## 2. 今後の課題（未対応）

| 項目 | リスク | 対策案 |
|---|---|---|
| レートリミット | SMS送信の連打による費用増加 | Firebase App Check の導入 |
| 画像の不適切コンテンツ | プロフィール写真に不適切な画像が投稿される | Gemini Vision API でのモデレーション |
| OP残高の競合 | 同時に複数承認でOP二重加算 | Firestoreトランザクションで対応済み |
| アカウント削除 | 退会時のデータ残存 | 削除APIと Firestore Cascadeの設計 |
| 不正メンバー | 迷惑行為ユーザーへの対処 | 通報・ブロック機能の追加 |

## 3. 環境変数一覧

本番環境（Vercel）と開発環境（.env.local）に設定が必要な変数。

### フロントエンド用（VITE_ プレフィックス）

| 変数名 | 説明 | 例 |
|---|---|---|
| VITE_FIREBASE_API_KEY | Firebase APIキー | AIzaSy... |
| VITE_FIREBASE_AUTH_DOMAIN | Firebase Auth ドメイン | owenweb-54f51.firebaseapp.com |
| VITE_FIREBASE_PROJECT_ID | Firebase プロジェクトID | owenweb-54f51 |
| VITE_FIREBASE_STORAGE_BUCKET | Firebase Storage バケット | owenweb-54f51.firebasestorage.app |
| VITE_FIREBASE_MESSAGING_SENDER_ID | Firebase メッセージング送信者ID | 139703842397 |
| VITE_FIREBASE_APP_ID | Firebase アプリID | 1:139703842397:web:... |
| VITE_FIREBASE_MEASUREMENT_ID | Firebase Analytics ID | G-5T2SGLPXKB |

### サーバーサイド用（Vercel Functions）

| 変数名 | 説明 | 設定場所 |
|---|---|---|
| FIREBASE_PROJECT_ID | Firebase プロジェクトID | Vercel 環境変数 |
| FIREBASE_CLIENT_EMAIL | サービスアカウントのメール | Vercel 環境変数 |
| FIREBASE_PRIVATE_KEY | サービスアカウントの秘密鍵 | Vercel 環境変数（改行注意） |
| GEMINI_API_KEY | Google AI Studio の APIキー | Vercel 環境変数 |

> **重要：** 上記の変数を `.env.local` に記述し、`.gitignore` に含めること。  
> `FIREBASE_PRIVATE_KEY` は改行を `\n` でエスケープして設定する。

## 4. .env.local テンプレート

```bash
# Firebase（フロントエンド）
VITE_FIREBASE_API_KEY=your_api_key_here
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.firebasestorage.app
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_FIREBASE_MEASUREMENT_ID=G-XXXXXXXXXX

# Firebase Admin SDK（Vercel Functions）
FIREBASE_PROJECT_ID=your_project_id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxx@your_project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nXXXXXX\n-----END PRIVATE KEY-----\n"

# Gemini API
GEMINI_API_KEY=your_gemini_api_key_here
```
