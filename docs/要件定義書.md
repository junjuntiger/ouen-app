# 要件定義書

**プロジェクト名：** おーえん（OUEN-APP）  
**作成日：** 2026-06-25  
**バージョン：** 1.0

---

## 1. 目的・背景

日本の伝統工芸・伝統芸能・地域文化を担う職人や演者は、
活動の継続に経済的困難を抱えるケースが多い。
一方、応援したいと考える人々は「どう支援すべきか分からない」という課題がある。

本アプリは、**現金での支払いに感謝の上乗せ（応援）を加える**という
シンプルな仕組みで、支援の文化を可視化・促進することを目的とする。

---

## 2. 対象ユーザー

### 受け取る側（伝統文化の担い手）
- 伝統工芸の職人・作家
- 伝統芸能の演者・師匠
- 地域文化の活動家・保存団体

### 支払う側（応援者）
- 日本文化を愛好する一般市民
- 伝統文化のファン・コレクター
- 地域コミュニティのメンバー

---

## 3. 機能要件

### 3-1. 認証機能
| ID | 機能 | 優先度 |
|---|---|---|
| AUTH-01 | 電話番号（SMS）でのログイン | 必須 |
| AUTH-02 | reCAPTCHA による不正アクセス防止 | 必須 |
| AUTH-03 | ログアウト | 必須 |
| AUTH-04 | ログイン状態の永続化 | 必須 |

### 3-2. プロフィール機能
| ID | 機能 | 優先度 |
|---|---|---|
| PROF-01 | 初回ログイン時のプロフィール登録 | 必須 |
| PROF-02 | 名前・職業・地域・ひとことの編集 | 必須 |
| PROF-03 | 定価メニューの登録・編集・削除 | 必須 |
| PROF-04 | プロフィール写真のアップロード | 今後追加 |

### 3-3. おーえん（取引）機能
| ID | 機能 | 優先度 |
|---|---|---|
| OUEN-01 | メンバー検索（名前・地域）での相手特定 | 必須 |
| OUEN-02 | QRコードスキャンでの相手特定 | 必須 |
| OUEN-03 | 定価メニューの選択 | 必須 |
| OUEN-04 | 支払い金額の入力 | 必須 |
| OUEN-05 | OPの自動計算（支払い − 定価） | 必須 |
| OUEN-06 | 取引内容の確認画面 | 必須 |
| OUEN-07 | 支払い側の送信（ペンディング状態作成） | 必須 |
| OUEN-08 | 受け取り側の承認・拒否 | 必須 |
| OUEN-09 | 承認後のOP残高への反映 | 必須 |

### 3-4. 表示・閲覧機能
| ID | 機能 | 優先度 |
|---|---|---|
| VIEW-01 | 自分のOP残高表示 | 必須 |
| VIEW-02 | おーえんタイムライン（全体・自分） | 必須 |
| VIEW-03 | 取引履歴の一覧 | 必須 |
| VIEW-04 | メンバー一覧・OP残高の表示 | 必須 |
| VIEW-05 | 承認待ちトランザクションの通知 | 必須 |

---

## 4. 非機能要件

| 項目 | 要件 |
|---|---|
| レスポンシブ | モバイルファースト（スマートフォン最適化） |
| パフォーマンス | 初回ロード 3秒以内（Vercel Edge配信） |
| セキュリティ | Firebase Security Rules で認証必須 |
| 可用性 | Firebase・Vercel のSLA準拠 |
| データ整合性 | Firestoreトランザクションによる原子的OP更新 |

---

## 5. 制約・除外事項

- OP は記録専用。購入・換金・移転はできない
- 実際の金銭決済機能は持たない（現金取引の記録のみ）
- マイナスOPは発生しない（支払い ≥ 定価のみ記録）
