## 2. users コレクション

**パス：** `users/{userId}`

**userId：** Firebase Authentication の uid を使用

### フィールド定義

| フィールド名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| uid | string | ✅ | Firebase AuthのUID | "abc123xyz" |
| name | string | ✅ | 表示名 | "山田 花子" |
| job | string | ✅ | 職業・活動内容 | "陶芸家" |
| area | string | ✅ | 活動地域 | "京都府" |
| message | string | | ひとこと（100文字以内） | "手びねりを教えています" |
| op | number | ✅ | OP残高（整数・0以上） | 12500 |
| menus | array | ✅ | 定価メニュー一覧（1件以上） | 下記参照 |
| photoURL | string | | プロフィール写真URL（Storage） | "https://..." |
| phone | string | ✅ | 電話番号（+81形式） | "+819012345678" |
| createdAt | timestamp | ✅ | アカウント作成日時 | Timestamp |
| updatedAt | timestamp | ✅ | 最終更新日時 | Timestamp |

### menus フィールド（配列の要素）

```json
{
  "name": "抹茶体験（60分）",
  "price": 5000
}
```

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| name | string | ✅ | メニュー名 |
| price | number | ✅ | 定価（円・整数・1以上） |

### サンプルドキュメント

## 3. transactions コレクション

**パス：** `transactions/{transactionId}`

**transactionId：** Firestoreの自動生成ID

### フィールド定義

| フィールド名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| fromUserId | string | ✅ | 支払い者のUID | "AbC123XyZ" |
| fromUserName | string | ✅ | 支払い者の名前（非正規化） | "田中 太郎" |
| toUserId | string | ✅ | 受け取り者のUID | "DeF456UvW" |
| toUserName | string | ✅ | 受け取り者の名前（非正規化） | "山田 花子" |
| menuName | string | ✅ | 選択したメニュー名 | "手びねり体験（60分）" |
| price | number | ✅ | 定価（円） | 5000 |
| paid | number | ✅ | 支払い金額（円・price以上） | 7000 |
| op | number | ✅ | 付与OP（paid - price） | 2000 |
| status | string | ✅ | 取引ステータス（下記参照） | "pending" |
| createdAt | timestamp | ✅ | 取引作成日時 | Timestamp |
| approvedAt | timestamp | | 承認日時 | Timestamp |

### status の値

| 値 | 説明 |
|---|---|
| pending | 支払い者が送信済み・受け取り者の承認待ち |
| approved | 受け取り者が承認済み・OP反映済み |
| rejected | 受け取り者が拒否 |

### サンプルドキュメント

```json
{
  "fromUserId": "AbC123XyZ",
  "fromUserName": "田中 太郎",
  "toUserId": "DeF456UvW",
  "toUserName": "山田 花子",
  "menuName": "手びねり体験（60分）",
  "price": 5000,
  "paid": 7000,
  "op": 2000,
  "status": "approved",
  "createdAt": "2026-06-25T10:00:00Z",
  "approvedAt": "2026-06-25T10:05:00Z"
}
```

## 4. Firestoreセキュリティルール

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // users: 本人のみ書き込み・全認証ユーザーが読み取り可
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;
    }

    // transactions: 関係者（fromUser/toUser）のみ操作可
    match /transactions/{transactionId} {
      allow read: if request.auth != null &&
        (resource.data.fromUserId == request.auth.uid ||
         resource.data.toUserId == request.auth.uid);

      // 作成：fromUserIdが自分であること
      allow create: if request.auth != null &&
        request.resource.data.fromUserId == request.auth.uid &&
        request.resource.data.paid >= request.resource.data.price;

      // 更新：toUserIdが自分（承認・拒否のみ）
      allow update: if request.auth != null &&
        resource.data.toUserId == request.auth.uid &&
        resource.data.status == "pending";
    }
  }
}
```

## 5. インデックス設計

Firestore複合インデックス（Firebase Consoleまたはfirestore.indexes.jsonで設定）

| コレクション | フィールド1 | フィールド2 | 用途 |
|---|---|---|---|
| transactions | toUserId (ASC) | createdAt (DESC) | 受け取り履歴の取得 |
| transactions | fromUserId (ASC) | createdAt (DESC) | 送付履歴の取得 |
| transactions | toUserId (ASC) | status (ASC) | 承認待ち取引の取得 |
| transactions | createdAt (DESC) | — | タイムライン取得 |
